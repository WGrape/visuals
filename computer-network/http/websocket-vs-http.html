🎯 核心区别总结

连接模式

短连接：请求-响应后立即关闭
长连接：多次请求-响应复用同一连接
WebSocket：握手后持久连接，任意时刻双向通信

通信方向

短/长连接：半双工，只能 Client 发起请求
WebSocket：全双工，双方平等，可主动发送

实时性

短/长连接：轮询或 SSE 实现伪实时
WebSocket：原生支持真正的实时通信

协议开销

HTTP：每次请求需完整 Header
WebSocket：首帧握手后，帧 Header 仅 2-14 字节

适用场景

RESTful API 表单提交 页面加载 低频数据获取 实时聊天 在线游戏 股票行情 协作编辑 IoT 设备状态 实时通知
