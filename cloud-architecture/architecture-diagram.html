📋 请求链路说明

① 客户端发起 HTTPS 请求到 api.test-example.com
② DNS 解析 CNAME 到云厂商 ELB 地址
③ SLB 负载均衡器接收请求，进行四层/七层转发
④ K8s Ingress Controller 根据路径规则路由到对应 Service
⑤ API Pod 处理业务逻辑，读写 MySQL 数据库
⑥ 热点数据写入 Redis 缓存，加速接口响应
⑦ 耗时任务异步投递到消息队列，由 Worker 处理
